# 2026-03-12 Notes

## Exa Brief 项目阶段性收口

### 项目目标
- 为 OpenClaw 构建两类日报/简报：
  - 技术简报（tech）
  - 社区热度简报（social）
- 初始目标是通过 Exa + LLM 生成高质量中文简报，并接入 cron + Feishu 群发。

### 关键演进路径
1. 早期直接用 prompt + cron + 模型生成，效果不稳定。
2. 发现仅靠 prompt 调模型，容易出现：
   - 搜索语义跑偏
   - 英文标题/摘要残留
   - 格式漂移
   - 多余结尾
3. 改为以官方 TS 思路为基线，在本地复现 Exa 检索链路。
4. 最终确定正式设计：
   - Exa 搜索
   - bucket 分层
   - 程序化 filter
   - 程序化 cluster
   - 程序化压缩为 final context
   - LLM 仅做最后成文
5. 后续再进一步收紧：
   - LLM 不再直接输出最终 markdown
   - 改为先输出结构化 JSON
   - 再由脚本统一渲染 markdown，解决格式/结尾漂移

### 当前正式实现
- skill 路径：`/root/.openclaw/workspace/ikoL/skills/exa-search`
- 正式脚本：`scripts/run_pipeline.ts`
- 支持两个 profile：
  - `tech`
  - `social`
- 支持输出目录：
  - tech 默认 `out/latest`
  - social 默认 `out/social-latest`
- pipeline 产物包括：
  - `01-raw.json`
  - `02-filtered.json`
  - `03-clustered.json`
  - `04-context-v3.json`
  - `04-context-v3.txt`
  - `05-final-report.json`
  - `05-final-report.md`

### 技术版（tech）特征
- 主源：GitHub / docs / X / V2EX
- 重点：PR / issue / regression / API / performance / architecture
- 输出标题：`📡 OpenClaw 技术简报`

### 社区版（social）特征
- 主源：X / Twitter / Weibo / V2EX / 掘金 / 少数派 / 36kr 等
- 重点：项目发布、社区讨论、用户反馈、生态扩展、市场 buzz
- 已额外加入高优先主题：
  - 互联网大厂 / AI 大厂 / 云厂商 / 企业协作平台对 OpenClaw 的实践应用
- 输出标题：`🌐 OpenClaw 社区热度简报`
- 当前判断：可用，但仍比 tech 更容易受媒体稿/教程稿风格影响；Loki 认可当前阶段先保持适度轻松风格，不继续深提纯。

### 已解决的关键问题
- 搜索语义跑偏：通过 bucket + filter + includeDomains / includeText 等方式收敛。
- 技术版英文标题残留：最终阶段强制中文标题化。
- social 无序号 / 漂格式：已改为结构化 JSON 输出后由脚本渲染 markdown。
- 结尾/格式失控：不再让 LLM 直接自由输出最终 markdown。

- 如果未来要让 `ikoL` 像调用参数化工具一样自然调用该 skill，则还需后续整理为：
