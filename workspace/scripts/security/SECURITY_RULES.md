# OpenClaw 安全行为规范

## 🚨 敏感命令黑名单

当OpenClaw需要执行以下敏感命令时，必须先向用户确认：

### 🔥 高危命令（必须确认）

#### 文件操作类
- `rm -rf` - 递归删除命令
- `rm` （删除重要文件时）
- `mv` （移动重要文件时）
- `dd` - 磁盘写入命令
- `mkfs.*` - 格式化文件系统
- `fdisk` - 磁盘分区操作

#### 系统配置类
- `ssh-keygen` - SSH密钥生成
- `ssh-copy-id` - SSH密钥复制
- `ssh` （连接到新主机时）
- `sudo chmod 777` - 危险的权限设置
- `chown root` - 更改为root所有权
- `usermod` - 用户权限修改

#### 网络配置类
- `iptables -F` - 清空防火墙规则
- `ufw disable` - 禁用防火墙
- `systemctl stop ssh` - 停止SSH服务
- `ifconfig` / `ip` （修改网络配置时）

#### 服务管理类
- `systemctl stop` - 停止系统服务
- `systemctl disable` - 禁用系统服务
- `kill -9` （终止重要进程时）

### ⚠️ 需要注意的文件路径

执行以下路径操作时需要特别小心：
- `/etc/` - 系统配置目录
- `/root/` - root用户家目录
- `/home/` - 用户家目录
- `/var/log/` - 系统日志目录
- `/usr/bin/` - 系统二进制文件
- `~/.ssh/` - SSH配置目录
- `~/.openclaw/` - OpenClaw配置目录

### ✅ 安全检查流程

1. **命令解析**：分析即将执行的命令
2. **风险评估**：检查是否涉及敏感操作
3. **用户确认**：如果涉及黑名单，必须获得用户明确同意
4. **操作记录**：记录所有敏感操作到审计日志

### 📝 确认模板

当需要用户确认时，使用以下格式：

```
🚨 安全确认请求

即将执行敏感命令：
命令：[具体命令]
风险等级：[高/中/低]
影响范围：[具体描述]
潜在后果：[可能的后果]

是否继续执行？请回复 "确认执行" 或 "取消"
```

### 🔍 例外情况

以下情况可以豁免确认：
- 在明确的安全沙箱环境中
- 用户在当前会话中已明确授权的重复操作
- 紧急故障恢复操作（但需要事后报告）

### 📋 审计记录

所有敏感操作都会记录到：
- 文件位置：`/var/log/openclaw/security-audit.log`
- 记录内容：时间、命令、用户确认状态、执行结果

---

**重要提醒**：这些规则旨在平衡安全性和效率，如有特殊情况需要调整规则，请与安全管理员协商。
