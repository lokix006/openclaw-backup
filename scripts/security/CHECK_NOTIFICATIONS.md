# 通知检查说明

## 自动检查机制

OpenClaw Assistant 会在每次对话开始时自动检查：

### 检查文件
- `/tmp/openclaw-notification-queue.txt` - 待发送的巡检通知

### 自动行为  
1. 检测到通知文件存在
2. 读取通知内容
3. 自动发送到用户飞书
4. 删除通知文件

### 时间流程
- **09:00** - 执行安全巡检
- **09:05** - 处理通知到队列
- **用户下次对话** - 自动发送通知

## 零Token消耗
- 不使用心跳机制
- 只在有实际通知时消耗token
- 估算：每天额外消耗 < 500 tokens
